### XOR

**說明:**

XOR運算式是用來計算兩個邏輯數值的差集。\n運算方式如下：\nTrue XOR True 傳回False\nTrue XOR False 傳回True\nFalse XOR True 傳回True\nFalse XOR False 傳回False

### Or

```xs
If Close >= Close[1] Or Close >= Close[2] Then ret = 1;
```

**說明:**

OR語法用來檢查運算式是否有任一個成立。\n在上述範例內如果Close欄位 >= 前期值 或是 Close欄位 >= 前兩期值的話，則ret會被設定成1。\n請參考AND語法。

### And

```xs
If Close >= Close[1] And Volume >= Volume[1] Then ret = 1;
```

**說明:**

AND語法用來檢查運算式是否同時成立。\n在上述範例內如果close欄位 >= 前期值 而且同時 volume欄位 >= 前期值的話，則ret會被設定成1。\n請參考OR語法。

### Not

```xs
If Close > Close[1] Then Ret = 1;
```

**說明:**

NOT語法回傳運算式的相反值。\n請看以下的範例程式:\n這個例子會在Close值大於Close的前期值時設定Ret為1。\n如果使用者希望的是在Close值 不是 大於Close的前期值時才設定Ret為1的話，則可以寫成:\nIf Not (Close > Close[1]) Then Ret = 1;\n上述的範例會在Close值 不是 大於Close的前期值時設定Ret為1。

### Cross Over

```xs
If Close Cross Above Average(Close, 5) Then ret = 1;
```

**說明:**

Cross相關的語法共有兩種：\nCross Above 或是 Cross Over 是用來檢查目前的欄位數值是否向上穿越某個欄位的前期數值。\nCross Below 或是 Cross Under 則是用來檢查目前的欄位數值是否向下跌破某個欄位的前期數值。\n以下是向上穿越均線的寫法：\n當這一期的Close欄位大於等於近5期的平均值(Average(Close,5))且前一期的Close欄位小於前一期的近5期的平均值的話，則ret會被設定成1。\n以下則是向下跌破均線的寫法：\nIf Close Cross Below Average(Close, 5) Then ret = 1;\n如果這一期的Close欄位小於等於近5期的平均值(Average(Close,5))且前一期的Close欄位大於前一期的近5期的平均值的話，則ret會被設定成1。\nCross也可以寫成Crosses。

### Begin End

```xs
If Close >= Close[1] Then ret = 1;  // 計算漲跌
```

**說明:**

Begin / End 語法用在 If, While, For 等控制指令內。當需要輸入超過一行的程式碼時，就必須使用Begin/End來把程式碼包圍起來。\n上述範例內當Close >= Close[1]時因為只需要執行一行指令，所以可以把指令直接寫出來。\n可是在以下的範例內，由於當Close >= Close[1]時我們希望要執行兩個指令，所以透過Begin/End把這兩個指令包圍起來。\nIf Close >= Close[1] Then\nBegin\n3   Value1 = Close - Close[1];  // 計算漲跌\n4   Value2 = Value1 / Close[1]; // 計算漲跌幅\nEnd;

### Ret

**說明:**

當警示腳本以及選股腳本在每根bar重新執行時 Ret的數值會被設定為0，當這根bar結束完成時如果Ret的數值不是0的話，則會產生觸發訊號(警示腳本)，或是選取這檔商品(選股腳本)。\nRet是一個系統的內建變數，他的數值會決定警示腳本以及選股腳本執行結果。\n以下是一個範例，在收盤價大於五日平均值時把Ret的數值設為1，用來觸發警示或是選取這個商品。\nIf Close > Average(Close, 5) Then Ret = 1;

### Break

```xs
i = 0;
```

**說明:**

Break指令的用處是控制迴圈執行時跳出迴圈的時機點，一般是用在For迴圈或是While迴圈內。\n以下是For迴圈的範例:\nFor i = 0 to 10\nBegin\n4	If Close[i] < 20 Then Break;\nEnd;\n一般而言上面的迴圈會執行11次(從I = 0 到 10)。可是在執行過程內，如果某一期的Close欄位值比20小的話，就會馬上跳出 For 迴圈。

### Repeat Until

**說明:**

Repeat/Until的語法是用來定義一段迴圈的執行邏輯。，語法如下:\nRepeat\n2  執行的指令;\nUntil 判斷式;\n程式會不斷的執行Repeat之後的指令，一直到Until後續的判斷式變成True值時才會離開迴圈。\n如果迴圈內需要執行的指令超過一個的話，則可以使用Begin/End來包圍:\nRepeat\n2  Begin\n3    執行的指令1;\n4    執行的指令2;\n5  End;\nUntil 判斷式;

**範例:**

```xs
SumValue = 0;
SumValue = SumValue + Close[i];
i = i + 1;
Until i = 4;
AvgValue = SumValue / 5;
```
Repeat
3  Begin
6  End;
上述範例內Repeat的迴圈會一直執行，每次執行時SumValue會累加前幾期的Close數值，同時變數 i 會每次加1。這個迴圈會一直跑到 i = 4 的時候才會離開。以這個範例而言，SumValue的數值會變成是最近５期收盤價的加總，最後算出AvgValue為最近５期的平均收盤價。
系統內還提供不同的迴圈控制方式，請參考While 以及 For。

### While

**說明:**

While語法是用來定義一段迴圈的執行邏輯。語法如下：\nWhile 判斷式\n2  執行的指令;\n當判斷式成立時，While迴圈會重複的執行，一直到判斷式回傳False為止。\n如果在迴圈內需要執行多個指令的話，則可以使用Begin/End的方式來包圍。\nWhile 判斷式\nBegin\n3  執行的指令1;\n4  執行的指令2;\nEnd;

**範例:**

```xs
SumValue = 0;
SumValue = SumValue + Close[i];
i = i + 1;
AvgValue = SumValue / 5;
```
While i < 5
3  Begin
6  End;
上述範例內While的迴圈會一直執行，直到 i 的數值 >= 5時才會停止。每次執行時SumValue會累加前幾期的Close數值，同時變數i 會每次加1。以這個範例而言，SumValue的數值會變成是最近５期收盤價的加總，最後算出AvgValue為最近５期的平均收盤價。
系統內還提供不同的迴圈控制方式，請參考Repeat/Until以及 For。

### For To

**For To (DownTo)**

```xs
SumValue = 0;
```

**說明:**

For語法是用來定義一段迴圈的執行邏輯。\nFor迴圈語法內必須使用一個變數，指定這個變數的初始值跟結束值，同時指定這個迴圈內要執行的指令:\nFor 變數 = 初始值 to 結束值\n2  執行的指令;\n如果要執行的指定超過一行的話則使用Begin/End語法來包裝需要執行的指定\nFor 變數 = 初始值 to 結束值\nBegin\n3  執行的指令1;\n4  執行的指令2;\nEnd;\n迴圈內的指令總共會被執行**(結束值 -  初始值 + 1)次，在期間每次執行時，變數的值會從初始值一一遞增到結束值**為止。\n以下是一個實例:\nFor i = 0 to 4\nBegin\n4	SumValue = SumValue + Close[i];\nEnd;\nAvgValue = SumValue / 5;\n上述的範例是一個累加的用法，透過For迴圈總共執行了5次(4 - 0 + 1)，第一次執行時i = 0(初始值), 第二次執行時i = 1(遞增), 最後一次執行時i = 4(結束值)。所以執行完For迴圈後SumValue的數值是最近５期Close欄位的累加值，把SumValue的值除以5之後就可以得到Close值的平均數值。\n如果迴圈的控制方式希望是從初始值一直減少直到結束值為止的話，則可以使用DownTo指令。\nSumValue = 0;\nFor i = 4 downto 0\nBegin\n4	SumValue = SumValue + Close[i];\nEnd;\nAvgValue = SumValue / 5;\n上述範例執行的結果與先前相同，唯一的差異是DownTo語法，所以迴圈執行的方式是第一次i = 4, 第二次 i = 3(遞減), 第三次 i = 2, 第四次 i = 1, 最後一次 i = 0。\n一般而言迴圈的執行次數是透過初始值跟結束值來控制的，可是如果需要在執行過程內提前跳出的話，則可以使用Break指令。\n系統內還提供不同的迴圈控制方式，請參考Repeat/Until以及 While。

### Once

```xs
Once(High = Highest(High, 5))
```

**說明:**

Once語法用來定義某些只需要執行一次的程式碼。\n舉例而言：\nBegin\n3    HighDate = Date;\n4    HighPrice = High;\nEnd;\nOnce語法之後必須填入一個判斷式，以上例而言，這個判斷式是 High = Highest(High, 5)，在判斷式之後，可以填入當判斷式成立時要執行的指令，如果有多行指令的話則可以使用Begin/End來包圍。\n所以上面這個範例執行的邏輯是，當創5日新高時，執行HighDate = Date，以及HighPrice = High這兩個指令，而且一旦出現創5日新高的情形之後，就不再執行HighDate = Date, 以及HighPrice = High這兩個指令。\n如果要達到同樣的目的，也可以使用IF指令，搭配一個紀錄是否曾經執行過的變數：\nVar: FirstTime(False);\nIf High = Highest(High, 5) And Not FirstTime Then\nBegin\n4    HighDate = Date;\n5    HighPrice = High;\n6    FirstTime = True;\nEnd;\n在上述範例內，程式使用FirstTime這個變數來紀錄這個IF狀態是否曾經發生過，以確保只會執行一次。\n可是由於系統會根據執行的設定方式在每一筆bar甚至每一筆tick更新時都會執行完整的程式碼，所以如果是使用If的寫法的話，每一次執行時還是會去判斷 High是否等於Highest(High, 5)！反之，如果是使用Once的寫法的話，一旦Once的運算式成立之後，未來不管執行任意bar，系統都會自動跳過Once的判斷式以及程式碼。由於在這個例子內，IF內所需要執行的指令比較複雜且費時，所以就可以使用Once的語法來提升執行的速度。

### Switch Case Default

**說明:**

### If Then Else

```xs
If Close > Open Then Ret = 1;
```

**說明:**

使用IF/THEN/ELSE這三個語法來判斷某個條件成立時該執行那個動作，不成立時又該執行那個動作。\n如果只需要判斷某個條件成立時該執行那個動作，則使用以下的語法:\n在上述範例內如果Close值大於Open值的話則 Ret 變數的數值會被設定為1。\n如果當條件成立時需要執行多個指令的話，則使用Begin/End的語法來包圍所需要執行的指令。\nIf Close > Open Then\nBegin\n3	Value1 = Close - Open;\n4	Value2 = High - Low;\nEnd;\n如果條件成立時跟不成立時都需要執行不同的指令的話，則可以加入 ELSE語法來定義條件不成立時該執行的動作。\nIf Close > Open Then\n2	Value1 = Close - Open\nElse\n4	Value1 = Open - Close;\n在上述範例內當Close的數值不大於Open的數值時，程式會執行Else內的語法。以這個例子為例，Value1的數值就是這根bar的實體高度。\n同樣的，Else之後也可以使用Begin/End語法來定義多個指令，範例如下:\nIf Close > Open Then\nBegin\n3	Value1 = Close - Open;\n4	Value2 = High - Low;\nEnd\nElse\nBegin\n8	Value1 = Open - Close;\n9	Value2 = High - Low;\nEnd;\nElse後面也可以接if，用else if來進行多層次的條件判斷，從腳本上至下依序縮小判斷範圍，範例如下:\nif value1 < 0 then\n2	value2 = 1\nelse if value1 < 10 then //等同於if  0 <= value1 and value1 < 10\n4	value2 = 2\nelse if value1 < 20 then //等同於if  0 <= value1 and value1 < 10\n6	value2 = 3\nelse  //等同於if  20<= value1\n8	value2 = 4;
